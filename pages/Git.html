Introduction

Git est un système de contrôle de version distribué, utilisé pour suivre les modifications apportées aux fichiers et coordonner le travail sur ces fichiers entre plusieurs personnes. Il a été créé par Linus Torvalds en 2005 pour le développement du noyau Linux.

Fonctionnalités Clés

  • Branches : Git permet de créer des branches pour travailler sur des fonctionnalités ou des correctifs de manière isolée. Les branches peuvent être fusionnées une fois le travail terminé.
  • Commits : Les commits sont des instantanés des modifications apportées aux fichiers. Chaque commit a un identifiant unique et un message descriptif.
  • Référentiels : Un référentiel (ou repo) est un dossier contenant tous les fichiers et l'historique des modifications. Les référentiels peuvent être locaux ou distants.
  • Fusion : La fusion (merge) permet de combiner les modifications de différentes branches.
  • Rebase : Le rebase permet de réappliquer les commits d'une branche sur une autre base, créant ainsi une histoire linéaire.

Commandes Git de Base

  • git init : Initialise un nouveau référentiel Git.
  • git clone : Clone un référentiel distant.
  • git add : Ajoute des fichiers à l'index (staging area).
  • git commit : Enregistre les modifications dans l'historique.
  • git status : Affiche l'état des fichiers dans le répertoire de travail et l'index.
  • git log : Affiche l'historique des commits.
  • git branch : Liste, crée ou supprime des branches.
  • git checkout : Change de branche ou restaure des fichiers.
  • git merge : Fusionne des branches.
  • git rebase : Réapplique les commits d'une branche sur une autre base.
  • git pull : Récupère et fusionne les modifications d'un référentiel distant.
  • git push : Envoie les modifications locales vers un référentiel distant.

Bonnes Pratiques

  • Commits Atomiques : Faites des commits atomiques, c'est-à-dire des commits qui représentent une seule modification logique.
  • Messages de Commit Clairs : Rédigez des messages de commit clairs et descriptifs.
  • Branches Thématiques : Utilisez des branches thématiques pour travailler sur des fonctionnalités ou des correctifs spécifiques.
  • Revue de Code : Effectuez des revues de code avant de fusionner des branches.
  • Tests : Testez les modifications avant de les valider.
