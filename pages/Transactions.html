Introduction

Les transactions sont un concept fondamental en gestion de bases de données. Elles garantissent la fiabilité, la cohérence et l'intégrité des données. Les principes ACID sont les piliers de la gestion des transactions et assurent que les modifications apportées à une base de données sont toujours correctes et durables.

Les différents types de transactions

  • Transactions courtes : Des transactions simples et rapides.
  • Transactions longues : Des transactions qui peuvent durer plusieurs minutes ou même des heures.
  • Transactions distribuées : Des transactions qui impliquent plusieurs bases de données.

Concepts plus spécifiques

  • Le verrouillage (locking) : Comment les systèmes de gestion de bases de données empêchent les conflits entre les transactions.
  • Les niveaux d'isolation : Les différents niveaux d'isolation possibles pour les transactions.
  • Les points de sauvegarde : Comment les bases de données sont sauvegardées pour assurer la récupération en cas de panne.

Les principes ACID

  • Atomicité : Une transaction est soit exécutée entièrement, soit annulée complètement. Il n'y a pas de demi-mesures.
  • Cohérence : Une transaction amène la base de données d'un état cohérent à un autre état cohérent. Les règles d'intégrité de la base de données sont toujours respectées.
  • Isolation : Les transactions sont exécutées de manière isolée, comme si elles étaient les seules à s'exécuter.
  • Durabilité : Une fois qu'une transaction a été validée, les modifications qu'elle a apportées sont permanentes, même en cas de panne système.
